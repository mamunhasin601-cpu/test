
# MVP: AI-карточки для маркетплейсов — лендинг + макет дашборда

Собираем визуальный MVP по плану из документа: продающий лендинг и каркас личного кабинета (без реальной AI-генерации и оплаты — только UI и мок-данные). Стиль — тёмный премиум/tech, две локали (RU по умолчанию, EN переключатель).

## Дизайн-направление

- **Палитра (dark premium):**
  - Фон: `#0A0A0F` (near-black), поверхности `#12121A`, границы `#1F1F2B`
  - Текст: `#F5F5F7` / muted `#8A8A99`
  - Акцент: электрический фиолетово-синий градиент `#7C3AED → #3B82F6`
  - Успех `#22C55E`, предупреждение `#F59E0B`
- **Типографика:** `Inter Tight` (заголовки, жирный, tight tracking) + `Inter` (текст). Крупные hero-заголовки 64–96px.
- **Ощущение:** мягкое свечение градиента за hero, тонкие границы, glass-карточки (`backdrop-blur`), микро-анимации hover, ambient-подсветка блоков.
- **Компоненты:** shadcn/ui в тёмной теме, скруглённые `rounded-2xl`, тени с фиолетовым оттенком.

## Структура маршрутов (TanStack Start)

```
src/routes/
  __root.tsx           # шелл + i18n провайдер + тема
  index.tsx            # лендинг
  dashboard.tsx        # layout кабинета (sidebar + Outlet)
  dashboard.index.tsx  # "Новая генерация"
  dashboard.cards.tsx  # "Мои карточки"
  dashboard.history.tsx
  dashboard.billing.tsx
  dashboard.settings.tsx
  login.tsx            # мок-форма (без backend)
```

Auth/бэкенд не подключаем — все данные мок-массивы. Кнопки «Войти/Попробовать бесплатно» ведут на `/dashboard`.

## Лендинг (`/`) — блоки

1. **Header:** лого «Карточная», навигация (Возможности · Галерея · Тарифы · Блог), переключатель RU/EN, кнопки «Войти» и «Попробовать бесплатно».
2. **Hero:** заголовок «Карточки и видео для WB и Ozon за 90 секунд», подзаголовок, 2 CTA, счётчик «12 480+ карточек создано», плавающий градиент-glow.
3. **Trust-полоса:** «112+ селлеров · ×2.1 CTR · без карты · первая карточка бесплатно».
4. **Инструменты (3 карточки):** Карточка-копировщик · Инфографика · Видеообложка 3–5 сек. У каждой — auto-play mock-превью (циклическая анимация/градиент вместо реального видео).
5. **Галерея примеров:** сетка карточек с фильтром по нишам (Одежда/Бьюти/Товарка/Электроника) — используем сгенерированные placeholder-изображения.
6. **Как это работает:** 3 шага (Загрузи фото → ИИ соберёт варианты → Скачай под маркетплейс).
7. **Тарифы:** 4 пакета кредитов (Мини/Стандарт/Про/Макс) с ценами и «хватит на N изображений / M видео».
8. **Финальный CTA:** «Первая карточка — бесплатно».
9. **SEO-текст:** длинный блок под коммерческие запросы («инфографика для Wildberries» и т.п.).
10. **Футер:** юр. лицо, оферта, политика, соцсети.

## Дашборд (`/dashboard`) — макет

- **Sidebar (240px, sticky):** Новая генерация · Мои карточки · История · Баланс · Настройки. Внизу — карточка баланса «250 кредитов» + кнопка «Пополнить».
- **Топбар:** название страницы, поиск, аватар.
- **Новая генерация:** большая dropzone «Перетащи фото товара» + альтернатива «Вставить ссылку на карточку конкурента» + селекторы (Тип: Карточка/Инфографика/Видео · Маркетплейс: WB/Ozon/Я.Маркет · Ниша) + кнопка «Сгенерировать (−15 кредитов)». После клика — мок-очередь со статусом (progress-bar с фейковой анимацией → 4 варианта-плейсхолдера).
- **Мои карточки:** grid из мок-карточек с hover-действиями (Скачать, Ещё вариант, Редактировать).
- **История:** таблица (дата · тип · маркетплейс · статус · кредиты).
- **Баланс:** текущие кредиты, таблица списаний, 4 пакета для покупки (мок).
- **Настройки:** данные компании, e-mail для чеков, API-ключ (заглушка).

## Интернационализация

- Лёгкий inline-провайдер: `src/lib/i18n.tsx` с объектами `ru`/`en` и хуком `useT()`. Хранение выбранного языка в `localStorage`. Без i18next, чтобы не раздувать зависимости.
- Все тексты лендинга и дашборда — через словари.

## SEO

- `head()` на каждом маршруте: уникальные title/description/og:title/og:description на RU.
- На лендинге — один `<h1>`, семантические `<section>`, alt на всех img.

## Что НЕ входит в этот шаг

- Реальная AI-генерация, fal.ai/OpenAI/Gemini интеграция.
- Auth, база данных, Lovable Cloud.
- Реальные платежи (ЮKassa/СБП).
- Парсинг карточек конкурентов, очереди воркеров.

Это следующая итерация — после подтверждения UI поднимем Lovable Cloud (auth + БД для генераций и кредитов) и подключим Lovable AI для генерации изображений.

## Технические детали

- **Стек:** TanStack Start (уже настроен), Tailwind v4, shadcn/ui, lucide-react иконки.
- **Шрифты:** `@fontsource-variable/inter` и `@fontsource/inter-tight` через `bun add`, импорт в `src/main.tsx`, подключение в `@theme` в `src/styles.css`.
- **Тема:** ставим `.dark` на `<html>` в `__root.tsx` (принудительная тёмная); переопределяем oklch-переменные в `src/styles.css` под новую палитру.
- **Плейсхолдер-визуалы:** 6–10 изображений карточек-примеров через `imagegen` (fast, jpg) в `src/assets/gallery/`.
- **Компоненты:** `LandingHero`, `TrustBar`, `ToolsGrid`, `Gallery`, `Pricing`, `Footer`, `DashboardSidebar`, `GenerationForm`, `MockQueue`, `CardsGrid`, `HistoryTable`, `BillingPanel`.
- **i18n:** `src/lib/i18n.tsx` (React context), словари `src/lib/i18n/ru.ts` и `.../en.ts`.
- **Валидация:** формы через `react-hook-form` + `zod` (уже в шаблоне).
- **Маршруты:** `dashboard.tsx` = layout с `<Outlet />`, остальные `dashboard.*.tsx` — вложенные страницы.
- **Head:** заменить дефолтные «Lovable App» в `__root.tsx` на реальные title/description проекта.
