# Концепция объединения двух сервисов

Черновик по итогам обсуждения. Цель — соединить функционал двух источников в один пайплайн
через общую промежуточную точку: **чистую фотографию товара**.

---

## Два источника

### Сервис 1 — «Копировщик» (по мотивам kartochka.me)
Что забираем:
- **Копирование** — пользователь даёт референс (ссылка на карточку конкурента или картинка),
  с которой хочет «скопировать» удачную сборку/композицию/стиль.
- **Очистку референса** — с чужой карточки снимаем инфографику и текст (чужие плашки,
  надписи, слоганы), оставляем чистую основу. Чужой контент не копируем — берём только
  структуру и подачу.
- **Предложение типов кадра** — как на сайте сервиса 1: на человеке (на модели),
  вид сверху и т.д. Сервис сам подсказывает подходящие сценарии.

### Сервис 2 — текущий код / логика obliqapp
Что забираем:
- **Фотографию, которую грузит пользователь** — его собственный товар.
- **Типы кадра** для этого фото: на человеке / вид сверху / вид снизу /
  в разложенном состоянии / в сложенном состоянии.

---

## Общий знаменатель обоих потоков

> **В обоих случаях нужна ЧИСТАЯ фотография товара.**

Это ключевая промежуточная сущность. Независимо от того, пришёл товар как загруженное
фото пользователя (сервис 2) или был извлечён/очищен с референса конкурента (сервис 1),
пайплайн сначала приводит всё к **чистому предметному фото товара без текста, плашек
и инфографики**, а уже из него собирает нужный тип кадра.

## Что происходит с текстом и инфографикой

- **Со входа (референс конкурента)** — текст и инфографика УДАЛЯЮТСЯ. Чужие надписи
  не копируем, берём только чистую основу.
- **На выходе (наша карточка)** — текст добавляется отдельно, ОПЦИОНАЛЬНО, только
  с разрешения пользователя. Сначала выдаём чистую картинку, текст — отдельным шагом.
- **Правила маркетплейсов сохраняются:** WB — текст на карточке разрешён;
  Ozon и Яндекс Маркет — текст на главном фото запрещён (там всегда чистый кадр).

### Два способа наложения текста (решение ещё не принято)
1. **Canvas-оверлей (код)** — идеальная кириллица, бесплатные правки, ограниченный дизайн.
   Подходит для простого заголовка/подзаголовка hero-карточки.
2. **gpt-image-2 (модель рисует текст)** — богатый дизайн, но риск опечаток и платные
   правки. Подходит для сложной инфографики (плашки, размерные сетки, сравнения).

Предварительный вывод: **гибрид** — canvas для простого текста hero-карточки,
gpt-image-2 для сложной инфографики.

## Задействованные AI-модели (текущие, через Lovable AI Gateway)

| Роль | Модель | Назначение |
|------|--------|-----------|
| Аналитик / сценарист | `openai/gpt-5-mini` | Vision: видит фото, описывает товар текстом |
| Художник (план v2) | `google/gemini-3.1-flash-image` (Nano Banana 2) | img2img: держит identity товара, переносит в сцену |
| Инфографика | `openai/gpt-image-2` | Кириллица поверх фото (стабильнее у этой модели) |

> ВАЖНО: планируется переезд на российский сервер — модели и endpoint'ы, вероятно,
> поменяются. Нужно уточнить, какой движок реально доступен и умеет ли он img2img
> (приём референсного изображения), inpainting (для очистки текста с референса).

## Ключевые технические задачи (вытекают из концепции)

1. **Шаг очистки референса** — новый этап: удаление текста/инфографики с чужой карточки
   (inpainting-модель или извлечение чистого товара). Раньше в пайплайне его не было.
2. **Приём референса по ссылке** — скачать картинку по URL и прогнать через тот же
   пайплайн. Сейчас на дашборде поля «вставить ссылку» нет, только dropzone.
3. **img2img движок** — чтобы художник реально видел фото товара, а не рисовал вслепую
   по текстовому описанию (текущее ограничение gpt-image-2 = text-only).
4. **Типы кадра как отдельное измерение** — presets поверх `PRESETS[marketplace]`:
   на модели / flat-lay (сверху) / снизу / разложенный / сложенный / packshot / деталь.
   Прятать несовместимые (например «на модели» для не-одежды).
5. **Возврат кредитов при неудаче** — обещание «бесплатные повторы при неудачной
   генерации» кодом сейчас не выполняется (кредиты списываются до генерации,
   при падении не возвращаются).
6. **Реальный прогресс вместо фейкового** — при стриминге img2img показывать
   партиалы с blur, а не таймерную крутилку.
